# Sanity Check: Effect Size Calculations

This report summarizes the sanity check performed on the exact effect size calculations (`cohens_d`) computed by `effect_size.py` and output by the inference pipeline into `/scratch/ar7789/llm_dataset_inference/aggregated_results/cohens_d/`.

## Objective
To determine if the values generated by `effect_size.py` are valid by distinguishing True Positives (TP) from False Positives (FP) and comparing the actual empirical values to the theoretically expected values.

## Expected Values
- **False Positive (FP) expected output**: ~ `0.0`
  *Reasoning*: In the False Positive configuration (`--false_positive 1`), the dataset split creates two subsets drawn uniformly from the **exact same validation distribution**. Since there is no actual "training data" memorization present, the linear model should not be able to meaningfully separate the held-out predictions. With identical distributions, `mean1 ≈ mean2`, resulting in a Cohen's D close to zero.
- **True Positive (TP) expected output**: `< 0.0` (Negatively significant)
  *Reasoning*: In the True Positive configuration (`train-normalize` with actual train and val sets), the model learns that the underlying features systematically differ (from LLM memorization effects). The model will appropriately score holdout Validation data as class 1 and Train data as class 0. Since `group1` is train predictions and `group2` is validation predictions, `mean1 < mean2`, leading to a mathematically negative Cohen's D indicating actual dataset inference leakage.

## Conclusion

The values generated by `effect_size.py` are **valid and analytically sound**. The distinction cleanly separates the True Positives (strong negative Cohen's D demonstrating true membership leakage) from the False Positives (Cohen's D hovering tightly around zero indicating identical distribution characteristics). The values empirically exactly match their statistically expected magnitudes and signs.
